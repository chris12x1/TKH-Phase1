SESSION 26: CLIENT-SIDE ATTACK PAYLOADS
==================================================

1. REFLECTED XSS
* Payload Used: <script>alert('XSS')</script>
* Concept: Why does this attack disappear when you refresh the page?
Answer: Reflected XSS is non persistent, Because the script is not stored in the application's database so when a refresh ouccurs it is sent without the payload.

2. STORED XSS (Cookie Theft)
* Cookie Value Stolen: session_id=admin_secret_99812_do_not_share 
* Payload Used: <script>alert(document.cookie)</script>


3. CROSS-SITE REQUEST FORGERY (CSRF)
* Malicious URL Crafted: http://127.0.0.1:8081/transfer?to=attacker&amount=5000
* Concept: What is the primary defense against CSRF?
Answer: The primary defense is the use of Anti-CSRF tokens. These are unique, cryptographically strong, and unpredictable tokens generated by the server for the user's session.
